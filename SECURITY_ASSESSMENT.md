# 보안성 자체 평가

## 회원 정보 유출 방지 방안

### 1. **비밀번호 암호화 (Password Hashing)**
- **SHA-512 해싱 알고리즘** 사용하여 비밀번호를 단방향 암호화
- 평문 비밀번호를 저장하지 않음
- 로그인 시 입력 비밀번호를 해싱하여 저장된 해시값과 비교

### 2. **JWT 기반 인증 토큰 시스템**
- **Access Token + Refresh Token** 이중 토큰 구조로 구성
- Access Token은 짧은 유효기간(15분), Refresh Token은 긴 유효기간(7일)
- 토큰 만료 시 자동 재발급 메커니즘

### 3. **토큰 검증 미들웨어**
- 모든 인증이 필요한 API에 토큰 검증 미들웨어 적용
- Bearer Token 형식으로 전송
- 토큰 서명 검증 및 만료 시간 체크
- 잘못된 토큰 또는 만료된 토큰 자동 차단

### 4. **환경 변수 관리**
- .env 파일을 통한 민감 정보 관리
- .gitignore에 환경 변수 파일 추가하여 Git 저장소에 노출 방지
- JWT_SECRET, DATABASE_URL, API 키 등 보안 정보 분리

### 5. **세션 관리**
- Express Session으로 소셜 로그인 세션 관리
- JWT_SECRET으로 세션 암호화
- resave: false, saveUninitialized: false로 불필요한 세션 저장 방지

### 6. **데이터베이스 보안**
- Prisma ORM을 통한 타입 안전한 쿼리 실행
- SQL Injection 공격 방지
- 데이터베이스 연결 정보는 환경 변수로 관리

### 7. **에러 처리**
- 상세한 에러 메시지 노출 방지
- 일관된 에러 응답 형식으로 정보 유출 최소화
- 중앙 집중식 에러 핸들러

## 보안 강점
✅ 비밀번호 단방향 해싱 적용  
✅ JWT 토큰 기반 인증 시스템  
✅ 환경 변수 분리 관리  
✅ SQL Injection 방지 (Prisma ORM)  
✅ 미들웨어 기반 토큰 검증 체계  

## 보안 개선 필요 사항
⚠️ CORS 설정이 현재 모든 origin(*) 허용 상태  
⚠️ HTTPS 미적용 시 secure 옵션 false 상태  
⚠️ 비밀번호 복잡도 검증 로직 부재
